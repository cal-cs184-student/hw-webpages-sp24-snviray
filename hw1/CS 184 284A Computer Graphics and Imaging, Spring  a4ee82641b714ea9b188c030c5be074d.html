Task 1

Walk through how you rasterize triangles in your own words.

  1. Find the minimum and maximum x and y values out of the given vertices to find the bounds of sampled pixels.
  1. I ran a nested for loop over all combinations of x and y pixel values between x_min and x_max and y_min and y_max
  1. I ran three line tests on x + 0.5 and y + 0.5 for each value of x and y. We add 0.5 to sample the middle of the pixel.
    • The line test is (x + 0.5 - x0) * (y1 - y0) + (y + 0.5 - y0) * (x1 - x0) >= 0 or V • N >= 0 where V is the vector formed by a pair of vertices of the triangle and N is the vector to normal V.
  1. If all three line tests outcomes are the same (all true or all false), that means the point is inside the triangle, so we will fill the pixel with the given color. If not, we won't render it.
    • I included the case where the test outcomes are all false to account for when the vertices are accounted in both winding orders.

My algorithm is no worse than one that checks each sample within the bounding box of the triangle because the for loop only loops over the bounding box of the triangle using the minimum and maximum x and y values of the vertices.

Task 2

Walk through your supersampling algorithm and data structures.

Data Structures:

  • sample_buffer
    • In order to support supersampling, I had to increase the size of the sample_buffer to represent an image with a larger resolution than the actual frame buffer.
    • Upon changing the sample_rate, the sample_buffer size was changed to height * width * sample_rate , where height and width are that of the framebuffer.
    • The sample buffer was mentally divided into sample_rate subsections, each of size height * width. Each subsection stored a sample for the corresponding point on the frame buffer.
    • For example, for sample_rate = 4, the 4 samples within point (x, y) can be accessed via these indices
      • sample_buffer[0 * width * height + y * width + x]
      • sample_buffer[1 * width * height + y * width + x]
      • sample_buffer[2 * width * height + y * width + x]
      • sample_buffer[3 * width * height + y * width + x]

Super Sampling Algorithm

rasterize_triangle

  1. I found the minimum and maximum x and y values out of the given vertices to find the bounds of sampled pixels.
  1. I ran a nested for loop over all combinations of x and y pixel values between x_min and x_max and y_min and y_max.
  1. For each pair of (x, y) pixel values, I ran another nested for loop to have counters spanning [0, sqrt(sample_rate)] for both the x and y pixel values. At each value, I calculated the actual x and y values in the middle for each sub-pixel using x_counter, y_counter, and the actual x and y values using:
    • float curr_x = (float) x_counter * increment + x + increment * (float) 0.5;
    • float curr_y = (float) y_counter * increment + y + increment * (float) 0.5;
  1. I ran three line tests on these actual points. If all three line tests outcomes are the same (all true or all false), that means the point is inside the triangle, so we will fill the pixel with the given color. If not, we won't render it.
  1. I stored the sample’s color value at the correct place in the sample_buffer, according to the indexing described in the previous section.

resolve_to_framebuffer

  1. I ran a nested for loop over all combinations of x and y pixel values between the width and height of the framebuffer.
  1. For each pair of (x, y) pixel values, I ran another for loop over counters spanning [0, sample_rate] to iterate over all the subsamples in the pixel.
  1. At each value, I indexed into the corresponding sample using the method described in the sample_buffer data structures section, and added the color of each sample for the pixel together.
  1. After running through sample_rate iterations for the pixel to get all the samples, I divided the sum of sample colors by 1 / sample_rate to get the average color for the pixel.
  1. I then wrote that average color into the frame buffer at (x, y) .

Why is supersampling useful? / Explain how you used supersampling to antialias your triangles.

According the Nyquist Theorem, we get no aliasing if no frequencies in the image are less than the half the sampling frequency. Supersampling is useful because it is a method of approximating the 1-pixel box-filter that serves as a lowpass filter. The 1-pixel box filter can be used to eliminate frequencies above half the sampling frequency. Therefore, we can achieve antialiasing with super sampling.

What modifications did you make to the rasterization pipeline in the process?

To support supersampling, I changed how the sample_buffer is resolved to the framebuffer because it was no longer a one to one relation. resolve_to_framebuffer now has translate between two resolutions to account for a changed sampling rate.

Explain barycentric coordinates in your own words and use an image to aid you in your explanation. One idea is to use a svg file that plots a single triangle with one red, one green, and one blue vertex, which should produce a smoothly blended color triangle.

Barycentric coordinates are a coordinate system that can represent all the points in a triangle in terms of the vertices of the triangle.

Each location inside the triangle is represented as a constants (α, β, γ) that represent a linear combination that expresses the relative distance of the point to each vertex in the triangle.

For example, if each vertex represented red, green, and blue. All of the coordinates would be some combination of red, green, and blue, with the weight of each color based on the distance from each vertex. They could all be represented as a linear combination of the colors, as shown in the picture.

Explain pixel sampling in your own words and describe how you implemented it to perform texture mapping.

Pixel sampling is used to map pixels to texels when texture mapping. I implemented it by:

  1. Converting the pixel value to be sampled to barycentric coordinates and then passing the coordinates to the sampling function as the uv vector with u representing x and v representing y.
    • I converted to barycentric coordinates using the equations in the lecture slides
  1. In the sample function, I scaled up the u value by the mip.width -1 and used mip.height - 1 for the v value.
  1. I decided how to convert the continuous (u, v) value to integer texel coordinates depending on the sampling method (see the section below).
  1. I used get_texel to get the texel for the corresponding integer coordinate(s) and linearly interpolated them together if necessary.

Briefly discuss the two different pixel sampling methods, nearest and bilinear.

Nearest takes the continuous pixel value and simply rounds to the nearest texel and samples the color located at that pixel.

Bilinear takes the continuous pixel values, finds the four nearest texel values (using floor and ceil on the continuous pixel values’ x and y coordinates) and linearly interpolates them together using their relative distances, and outputs the combined result.

Use the pixel inspector to find a good example of where bilinear sampling clearly defeats nearest sampling.

Comment on the relative differences. Discuss when there will be a large difference between the two methods and why.

The edges in the bilinear sampling pictures look a bit smoother than nearest sampling, but as the sampling rate increases the difference is less prominent.

There will be a larger difference between the two methods in textures with higher frequencies/two very different colors next to each other because bilinear sampling averages colors together and will smooth out harsh contrast.

Explain level sampling in your own words.

Level sampling involves storing multiple versions of the same image at different resolutions as “levels.” When actually sampling textures to create an image, each sample would sample from the mipmap with a resolution closest to screen sampling rate of the actual size of the “footprint” that the texture sample will fill.

Describe how you implemented level sampling for texture mapping

  1. I implemented get_level, which found a continuous value for the appropriate mipmap level by estimating the actual size of the texture footprint using coordinates of neighboring samples (x + 1, y) and (x, y + 1) as described in lecture slides.
    1. I clamped the levels between 0 and mipmap.size() - 1
    1. I ran the line test on neighboring samples, and if they were outside of the triangle I clamped the neighboring values passed into get_level to just be (x, y) .
  1. After retrieving the level, for level_nearest, I just rounded the continuous value and sampled from that rounded level mipmap. For linear level sampling, I used floor and ceil to find and sample from the two closest mipmaps and linearly interpolated the samples’ colors together using the fractional component of the continuous level as the x constant in lerp(x, v0, v1) .

Describe the tradeoffs between speed, memory usage, and antialiasing power between the three various techniques.

Pixel Sampling

  • Speed: Bilinear sampling needs to process more texels per pixel (in my implementation, 4 texels per pixel) so its a bit slower than simple rasterization. But nearest sampling does not really impact speed as much compared to simple rasterization because all it only requires one access into the texture map per pixel.
  • Memory usage: Uses more memory than simple rasterization because need to store texture image.
  • Antialiasing Power: Bilinear sampling, similar to sampling per pixel sampling, can smooth jaggies.

Level Sampling

  • Speed: Faster than supersampling because it has less computation and can utilize texture caching
  • Memory usage: Uses more memory than pixel sampling because we need to store the same texture image at multiple resolutions. However, it is stored in a hierarchy for efficiency.
  • Antialiasing Power: removes aliasing caused by texture minification (when there is only one pixel sample that corresponds to multiple texel samples)

Sampling per Pixel Sampling

  • Speed: slower, have to iterate over all the subsamples at sample_rate times per pixel.
  • Memory usage: Uses sample_rate times more memory than only sampling once per pixel
  • Antialiasing Power: Removes jaggies by effectively sampling at more than twice the signal frequency
